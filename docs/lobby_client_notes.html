Lobby Client Notes

Chris Greenhalgh, 2010-09-16

Notes

Difference deployment contexts:

  • browser-only lobby client
    • cannot open market URLs on Android
    • cannot send intents on Android
    • cannot check app is installed
    • can loadUrl for URL (which may not exist) with custom scheme - starts already-installed app with suitable registration on Android or iOS
    • can loadUrl for URL (which does exist and is accessible) with custom MIME type - starts already installed app with suitable registration on Android after connecting to server to get ContentType (iOS unknown)
  • game-specific embedded client
    • index should be local asset
      • lobbyclient.getGame().getIndexJson()
    • queryUrl should be for server
      • lobbyclient.getGame().getQueryUrl()
    • URL interception can start real client
    • real client can pass version info to JS 
      • lobbyclient.getGame().getAppId(), appMajorVersion, appMinorVersion, clientName
    • [needs localStorage (or equiv.) providing]
  • generic lobby client [standalone or embedded within generic game client]
    • requires index interaction
    • may push to market for app installation
    • requires native helper to check app installation [possible on Android at least]
